TypeScript: Week 1

Learning Goals

  • Understand what TypeScript is and why we use it
  • Introduce the concept of types in programming languages
  • See how TypeScript enhances variables, functions, arrays, and objects with types
  • Learn how TypeScript handles conditionals, logical operators, and iteration
  • Discover additional TypeScript features that help prevent bugs and improve code clarity

Introduction

By now, you've learned the basics of JavaScript — variables, functions, arrays, objects, and more. In this lesson, we'll explore TypeScript, a superset of JavaScript that adds types to our code.

Why does this matter? Types help us catch mistakes earlier, write clearer code, and build more robust programs. When we say that TypeScript is a typed superset of JavaScript, we mean that it brings type safety — protection against accidentally using the wrong kinds of data — and compiler errors that tell us about mistakes before the code even runs. Don't worry — if you know JavaScript, you're already most of the way there. Let's dive in!

Side Note: What is a compiler? A compiler is a tool that takes the code you write and converts it into something the computer can run. JavaScript doesn't need a compiler — browsers understand it directly. TypeScript, however, needs to be compiled into JavaScript before it can run in a browser. The TypeScript compiler checks your code for errors and turns it into regular JavaScript. You don't have to worry about the compiler doing anything fancy behind your back — it's just there to help catch mistakes!

Bonus Benefit: Thanks to TypeScript's types, many code editors (like Visual Studio Code) can give you autocomplete suggestions, documentation tooltips, and real-time error hints as you type. This is called editor tooling support — and it can make you faster and more confident as a developer!

What is a Type?

A type tells the computer what kind of data a variable can hold. For example:

  • number → numbers like 42
  • string → text like "hello"
  • boolean → true or false

Top Tip: You don't always need to specify a type — TypeScript is smart and can figure it out (called type inference). But adding types explicitly can make your code clearer!

Variables with Types

Declaring variables in TypeScript is almost identical to JavaScript, but we can optionally specify the type:

let count: number = 42;
const name: string = "Ada";
let isActive: boolean = true;

Without types (TypeScript infers them automatically):

let count = 42;  // TypeScript knows this is a number
const name = "Ada";  // TypeScript knows this is a string

Important: Once a variable has been assigned a type (either explicitly or through inference), you can't reassign it to a different type later. For example:

let score = 100; // inferred as number
score = "high"; // ❌ Error: Type 'string' is not assignable to type 'number'

Optional Properties and Default Values

Sometimes a property or parameter might not always be provided. We can mark it as optional by adding a ?:

type Person = {
  name: string;
  age?: number; // optional property
};

const person1: Person = { name: "Ada" };
const person2: Person = { name: "Grace", age: 40 };

For function parameters, we can also provide default values:

function greet(name: string = "friend"): string {
  return `Hello, ${name}!`;
}

console.log(greet()); // "Hello, friend!"
console.log(greet("Alan")); // "Hello, Alan!"

Clarification: Optional properties and default values help make your code more flexible and reduce the need for complex conditional checks.

Strictness Settings

TypeScript has different strictness settings that control how careful it is when checking your code. The most common setting is strict mode, which turns on several useful checks:

  • Warns you if a variable could be null or undefined
  • Ensures all functions return the correct type
  • Prevents using variables before they are assigned

Top Tip: Most developers recommend enabling strict mode for maximum safety. You'll thank yourself later!

Conditionals and Logical Operators

Same as JavaScript — but with more confidence that the values are what we expect!

const age: number = 18;

if (age >= 18) {
  console.log("You can vote!");
} else {
  console.log("You are too young to vote.");
}

Logical operators work the same:

const isStudent: boolean = true;
const hasID: boolean = false;

if (isStudent && hasID) {
  console.log("You get a discount!");
}

Clarification: Since TypeScript knows the types, it can warn us if we accidentally compare mismatched types. For example, comparing a string to a number might cause an error or warning, helping you catch bugs faster.

Functions with Types

Functions can have types for both their parameters and return values:

function greet(name: string): string {
  return `Hello, ${name}!`;
}

console.log(greet("Grace"));
// => "Hello, Grace!"

Clarification: In JavaScript, you could accidentally call a function with the wrong type of argument (for example, greet(42)) and not see an error until your code runs. In TypeScript, you'll get an error right away:

greet(42); // ❌ Error: Argument of type 'number' is not assignable to parameter of type 'string'

Bonus Benefit: When you define types on function parameters, your editor can show function signatures (little popups reminding you of the expected parameters and return types) as you type. This can save you trips to the documentation and make your coding flow smoother.

Arrays and Objects with Types

Arrays

We can specify what type of values the array holds:

const numbers: number[] = [1, 2, 3, 4];
const names: string[] = ["Ada", "Grace", "Linus"];

In JavaScript, arrays can contain values of different types (for example: [1, "hello", true]). TypeScript encourages us to be more precise. If we want an array with mixed types, we can explicitly say so:

const mixed: (number | string | boolean)[] = [1, "hello", true];

Top Tip: It's best to keep arrays consistent in their types whenever possible. It makes your code easier to reason about!

Objects

We define object types to specify the shape of an object:

type Person = {
  name: string;
  age: number;
};

const person: Person = {
  name: "Alan",
  age: 35
};

Clarification: In JavaScript, you could accidentally forget a property or add an unexpected one to an object. TypeScript will give you an error if you forget a required property or add something that doesn't belong:

const person: Person = {
  name: "Alan"
}; // ❌ Error: Property 'age' is missing

const person: Person = {
  name: "Alan",
  age: 35,
  job: "engineer"
}; // ❌ Error: Object literal may only specify known properties

Bonus Benefit: When you define object types, your editor can show autocomplete suggestions for property names as you type. This makes it easier to remember what properties exist and reduces typos.

Iteration

Loops work exactly the same — and TypeScript knows the type of each value inside the loop!

const fruits: string[] = ["apple", "banana", "cherry"];

for (const fruit of fruits) {
  console.log(fruit);
}

Clarification: Because TypeScript knows that fruit is a string, it can warn you if you accidentally try to do something invalid on it (like calling a number method on it).

Conclusion

TypeScript builds on everything you've learned about JavaScript, adding the power of types to make your code safer and clearer. Thanks to type safety, compiler errors, and rich editor tooling, TypeScript helps catch bugs early, speeds up your coding workflow, and gives you confidence that your code works the way you expect. You've now seen how types apply to variables, functions, arrays, objects, and more — along with helpful features like optional properties, default values, and strictness settings. As you keep learning, you'll discover even more ways TypeScript helps you write great code!
